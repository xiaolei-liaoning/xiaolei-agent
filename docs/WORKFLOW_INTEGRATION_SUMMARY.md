- ✅ **性能优化**: 按需加载，不使用时不占用资源
- ✅ **易于维护**: 两个模块可以独立开发和更新

### 2. 后端集成 - 添加路由支持

#### 2.1 创建工作流编辑器路由
**文件**: `main.py`  
**位置**: coze_page路由之后（约第343行）

**代码实现**:
```python
@app.get("/workflow_editor", response_class=HTMLResponse, summary="智能工作流编辑器")
async def workflow_editor_page():
    """提供工作流编辑器页面"""
    templates_dir = Path(__file__).parent / "templates"
    workflow_html = templates_dir / "workflow_editor.html"
    if workflow_html.exists():
        with open(workflow_html, "r", encoding="utf-8") as f:
            return f.read()
    return "<html><body><h1>工作流编辑器文件不存在</h1></body></html>"
```

**路由映射**:
- `/coze` → Coze低代码平台主页
- `/workflow_editor` → 智能工作流编辑器（iframe源）

         ↓ HTTP请求

### 数据流向

1. **用户操作**: 点击左侧导航栏的"工作流"标签
2. **前端响应**: JavaScript显示`#workflow`模块，隐藏其他模块
3. **iframe加载**: 浏览器请求`/workflow_editor`端点
4. **后端响应**: FastAPI返回workflow_editor.html内容
5. **渲染完成**: 工作流编辑器在iframe中完全加载

## 🔧 使用指南

### 访问工作流编辑器

**方式1: 通过Coze平台**
1. 打开浏览器访问: http://localhost:8001/coze
2. 点击左侧导航栏的"🕸️ 工作流"标签
3. 工作流编辑器将在右侧区域加载

**方式2: 直接访问**
- 访问: http://localhost:8001/workflow_editor

### 基本操作流程

1. **创建节点**: 从左侧节点库拖拽节点到画布
2. **连接节点**: 点击节点的连接点，拖拽到目标节点
3. **配置属性**: 点击节点，在右侧面板编辑配置
4. **测试执行**: 点击底部工具栏的"▶️ 测试执行"按钮
5. **保存工作流**: 点击右上角的"保存"按钮

### 循环节点使用示例

1. 拖拽"🔄 循环节点"到画布
2. 系统自动生成下方的工作流框
3. 在工作流框内添加处理节点（如LLM）
4. 自动建立循环连接（进入→处理→返回）
5. 配置循环参数（最大迭代次数、循环类型）
