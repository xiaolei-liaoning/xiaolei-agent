## 📋 架构分层详解

### 第1层：客户端层（Client Layer）

**职责：** 提供多样化的用户接入方式

#### 支持平台
| 平台 | 技术栈 | 特点 |
|------|--------|------|
| **Web前端** | Vue.js / React | 响应式设计，跨浏览器兼容 |
| **微信小程序** | UniApp | 微信生态内无缝集成 |
| **桌面应用** | Electron | 跨平台桌面体验 |
| **移动App** | iOS/Android原生 | 高性能移动端应用 |
| **API调用方** | RESTful API | 第三方系统集成 |

#### 关键特性
- ✅ 统一RESTful API接口
- ✅ WebSocket实时通信
- ✅ 多端数据同步
- ✅ 统一的身份认证

---

### 第2层：统一API层（Unified API Layer）

**职责：** 标准化接口服务，版本控制

#### API分类
| 接口类型 | 路径 | 方法 | 说明 |
|---------|------|------|------|
| **聊天接口** | `/api/v1/chat` | POST | 发送消息、获取回复 |
| **任务调度** | `/api/v1/tasks` | POST/GET | 提交任务、查询状态 |
| **技能管理** | `/api/v1/skills` | GET | 获取可用技能列表 |
| **监控接口** | `/api/v1/monitor` | GET | 系统监控信息 |
| **工作流引擎** | `/api/v1/workflow` | POST/GET | 创建工作流、执行 |

#### 设计原则
- ✅ RESTful规范
- ✅ 版本控制（v1/v2）
- ✅ 统一的错误码
- ✅ OpenAPI/Swagger文档

---

### 第3层：智能调度中心（Intelligent Scheduler）

**职责：** 智能任务分配和路由决策

#### 五步调度流程
```
①意图识别 → ②任务拆解 → ③Agent路由 → ④负载均衡 → ⑤任务分发
```

#### 核心组件

**1. IntentRecognizer（意图识别器）**
- 基于关键词匹配
- 置信度计算
- 多意图识别

**2. TaskDecomposer（任务拆解器）**
- **规则引擎**：快速匹配（置信度≥0.6）
- **LLM路径**：GLM-4-flash智能分解
- **降级机制**：规则失败自动切换LLM

**3. AgentRouter（Agent路由器）**
- **评分公式**：
  ```
  评分 = 优先级×0.3 + 健康度×0.25 + (1/执行时间)×0.2 + 成功率×0.25
  ```
- 四维加权模型
- 动态评分更新

**4. LoadBalancer（负载均衡器）**
- 避免单点过载
- 故障隔离
- 自动降级

### 第5层：核心支撑层（Core Support Layer）

**职责：** 提供核心业务能力和算法支持

#### 第一组：AI能力组件（5个）

**1. 深度思考引擎**
- 推理引擎：多轮隐式推理
- 自主搜索：SerpAPI + DuckDuckGo
- 自我反思：答案质量验证
- 实现位置：`core/reasoning_engine.py`

**2. RAG搜索引擎**
- 主题搜索：按知识主题分类
- 知识摘要：自动生成摘要
- 增强过滤：多维度筛选
- 智能排序：相关性+权威性+时效性
- 实现位置：`core/rag_search_engine.py`

**3. 向量存储系统**
- 定时备份：每24小时自动备份
- 内存优化：批量写入缓冲（10条或30秒）
- 恢复机制：一键数据恢复
- 自动清理：保留最近7天
- 实现位置：`core/vector_store_manager.py`

**4. BFS处理工具**
- 文本BFS遍历：广度优先搜索
- 内容树处理：层次化结构处理
- 全局工具类：任意模块可调用
- 实现位置：`core/bfs_processor.py`

**5. 关键词提取器**
- TF-IDF算法：统计方法
- TextRank算法：图算法
- BM25算法：概率模型
- LLM语义理解：GLM-4辅助
- 实现位置：`core/keyword_extractor.py`

#### 第二组：业务组件（5个）

**6. 对话压缩系统**
- 短期压缩：最近10轮对话
- 中期压缩：最近50轮对话
- 长期压缩：最近30天对话
- 向量检索：语义相似度搜索
- 实现位置：`core/conversation_compressor.py`

**7. 工作流引擎**
- XML解析：可视化工作流定义
- 节点执行：LLM/Tool/Condition/Loop/Parallel
- 条件分支：动态路由
- 循环节点：批量处理
- 实现位置：`skills/workflow_engine.py`

**8. GUI自动化**
- PyAutoGUI：鼠标键盘控制
- 打开/关闭应用
- 截图功能
- 文本输入
- 实现位置：`skills/gui_automation/handler.py`

**9. 数据分析引擎**
- Pandas：数据处理
- Matplotlib：图表生成
- Scikit-learn：机器学习
- 可视化报表
- 实现位置：`skills/data_analysis/handler.py`

**10. Web爬虫引擎**
- Playwright：浏览器自动化
- BeautifulSoup：HTML解析
- 热搜抓取：微博/百度/B站/抖音
- 内容提取：结构化数据
- 实现位置：`skills/web_scraper/handler.py`

### 第6层：基础设施层（Infrastructure Layer）

**职责：** 提供底层技术支撑

#### 核心服务（6大基础服务）

**1. LLM后端**
- GLM-4主模型：高质量回答
- GLM-4-flash：快速响应
- 超时重试：30秒超时，指数退避
- 降级策略：模型不可用时切换
- 实现位置：`core/llm_backend.py`

**2. Redis缓存**
- 会话缓存：用户对话历史
- 任务状态：实时任务进度
- 技能缓存：频繁使用的结果
- TTL管理：自动过期清理
- 配置：`REDIS_URL`, `REDIS_CACHE_TTL=3600`

**3. ChromaDB**
- 向量索引：高效语义检索
- 批量写入：减少I/O压力
- 备份恢复：数据安全
- 内存优化：延迟加载
- 实现位置：`core/vector_store.py`

**4. 数据库**
- SQLAlchemy：ORM框架
- SQLite：轻量级存储
- 数据持久化：长期保存
- 事务管理：数据一致性

**5. 日志系统**
- Loguru：现代化日志库
- 分级日志：DEBUG/INFO/WARNING/ERROR
- 性能监控：关键指标记录
- 告警通知：异常情况推送

**6. 辅助服务**
- **消息队列**：asyncio异步通信
- **配置管理**：.env文件 + YAML配置
- **监控系统**：psutil资源监控
- **安全模块**：输入验证 + API鉴权 + 速率限制
- **工具库**：jieba分词 + networkx图算法 + sklearn机器学习

## 🎯 架构特点总结

### ✅ 分层清晰
- **6层架构**：职责明确，易于维护
- **单向依赖**：上层依赖下层，避免循环依赖
- **接口标准化**：每层都有清晰的接口定义

### ✅ 高度解耦
- **前后端分离**：客户端与后端完全解耦
- **模块独立**：每个组件可独立开发和测试
- **插件化设计**：技能、人物SKILL可动态加载

### ✅ 可扩展性
- **横向扩展**：每层都可水平扩展
- **Agent扩展**：轻松添加新Agent类型
- **技能扩展**：简单注册即可添加新技能

### ✅ 高可用性
- **降级机制**：规则→LLM→默认回复
- **故障隔离**：单个Agent故障不影响整体
- **自动重试**：网络错误自动重试3次
- **数据备份**：向量数据定时备份

### ✅ 高性能
- **异步处理**：asyncio异步IO
- **并发执行**：23个并发线程（5+10+5+3）
- **缓存优化**：Redis多级缓存
- **批量操作**：向量写入批量flush

### ✅ 智能化
- **AI驱动**：任务拆解、路由决策
- **语义理解**：关键词提取、意图识别
- **自主学习**：RAG知识检索
- **深度思考**：多轮推理+自我反思

### ✅ 可观测性
- **完善监控**：CPU、内存、网络实时监控
- **详细日志**：分级日志记录
- **告警系统**：阈值触发告警
- **性能指标**：任务成功率、响应时间
