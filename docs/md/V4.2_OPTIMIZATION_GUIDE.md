## 1. 任务拆解优化：规则兜底 + LLM智能泛化

### 1.1 架构设计

```
用户输入 → 意图识别 → 规则引擎(置信度≥0.6) → 成功? → 执行
                                    ↓ 失败
                              LLM智能拆解(GLM-4-flash)
                                    ↓
                              子任务并行执行
```

### 1.2 核心特性

✅ **双层策略**
- 规则引擎快速匹配（简单任务）
- LLM语义理解泛化（复杂任务）

✅ **规则引擎**
- 基于关键词权重的快速任务分解
- 置信度阈值：0.6
- 响应时间：<50ms

✅ **LLM路径**
- 使用GLM-4-flash进行智能分解
- 支持动态参数提取
- 适应性强，可处理未见过的任务类型

✅ **降级机制**
- 规则失败自动切换到LLM路径
- 保证系统100%可用性

✅ **并行执行**
- 子任务可并行处理
- 提高整体效率30-50%

### 1.3 实现位置

**文件：** `core/task_decomposer.py`

### 1.4 使用示例

```python
from core.task_decomposer import TaskDecomposer

decomposer = TaskDecomposer()

# 简单任务 - 规则引擎处理
tasks = decomposer.decompose("查询北京天气")
# 输出: [{"type": "weather", "params": {"city": "北京"}}]

# 复杂任务 - LLM智能拆解
tasks = decomposer.decompose("研究人工智能发展趋势并生成报告")
# 输出: [
#   {"type": "search", "params": {"query": "人工智能最新发展"}},
#   {"type": "scrape", "params": {"url": "$search_result"}},
#   {"type": "analyze", "params": {"data": "$scraped_content"}},
#   {"type": "summarize", "params": {"text": "$analysis_result"}}
# ]
```

## 2. Agent路由优化：多维加权路由模型

### 2.1 评分公式

```
Agent评分 = 优先级×0.3 + 健康度×0.25 + (1/执行时间)×0.2 + 成功率×0.25
```

### 2.2 核心特性

✅ **多维加权模型**
综合考虑4个维度：
- **优先级** (权重0.3)：任务紧急程度
- **服务健康度** (权重0.25)：Agent当前负载和可用性
- **执行时间** (权重0.2)：历史平均响应速度
- **成功率** (权重0.25)：任务执行成功概率

✅ **动态评分**
- 实时计算Agent可用性和性能指标
- 每5秒更新一次评分

✅ **负载均衡**
- 避免单点过载
- 自动选择最优Agent

✅ **故障隔离**
- 异常Agent自动降权
- 保障系统稳定性

### 2.3 实现位置

**文件：** `core/multi_agent_system.py`  
**方法：** `_select_optimal_agent()`

### 2.4 监控示例

```python
# 获取Agent路由评分
agent_scores = await agent_scheduler.get_agent_scores()
for agent_type, score in agent_scores.items():
    print(f"{agent_type}: {score:.2f}")
    # 输出示例:
    # check: 0.85
    # scrape: 0.92
    # vulnerability: 0.78
    # summarize: 0.88
```

## 3. 向量存储优化：轻量化数据运维

### 3.1 核心功能

✅ **定时备份**
- 每24小时自动备份
- 备份路径：`~/.小雷版小龙虾/backups`
- 压缩存储，节省空间

✅ **内存优化**
- 批量写入缓冲
- 触发条件：10条记录 或 30秒
- 减少I/O压力

✅ **恢复机制**
- 支持从备份文件一键恢复
- 数据完整性校验

✅ **自动清理**
- 定期清理过期备份
- 默认保留最近7天

### 3.2 备份策略

```
每日凌晨2:00 → 全量备份 → 压缩存储 → 保留最近7天
                                    ↓
                              备份目录结构:
                              backups/
                                ├── 2026-04-26_02-00-00.chroma
                                ├── 2026-04-25_02-00-00.chroma
                                └── ...
```

### 3.3 实现位置

**文件：** `core/vector_store_manager.py`

### 3.4 使用方法

```python
from core.vector_store_manager import VectorStoreManager

manager = VectorStoreManager()

# 手动触发备份
backup_path = manager.create_backup()
print(f"备份完成: {backup_path}")

# 从备份恢复
manager.restore_from_backup(backup_path)
print("恢复完成")

# 清理旧备份（保留最近7天）
manager.cleanup_old_backups(days=7)
```

## 4. BFS工具泛化：全局文本处理能力

### 4.1 核心功能

✅ **全局工具类**
- `BFSTextProcessor` 提供统一的BFS文本处理接口
- 可在任意模块调用

✅ **双功能支持**
- **文本BFS遍历**：广度优先搜索文本内容
- **内容树处理**：构建和处理层次化内容结构

✅ **独立模块**
- 文件：`core/bfs_processor.py`
- 零依赖，易于集成

### 4.2 实现位置

**文件：** `core/bfs_processor.py`

### 4.3 使用示例

```python
from core.bfs_processor import BFSTextProcessor

processor = BFSTextProcessor()

# 文本BFS遍历
result = processor.traverse_text(
    text="这是一段长文本...",
    max_depth=3,
    callback=lambda node: print(node)
)

# 内容树处理
tree = processor.build_content_tree(document)
processed = processor.process_tree(tree, strategy="breadth_first")
```

## 5. 终端适配优化：前后端完全解耦

### 5.1 核心改进

✅ **统一API前缀**
- 所有接口统一使用 `/api/v1` 前缀
- 清晰的版本标识

✅ **RESTful规范**
- 标准的HTTP方法（GET/POST/PUT/DELETE）
- 标准的HTTP状态码（200/400/404/500等）

✅ **版本控制**
- 支持v1/v2多版本并存
- 平滑升级，不影响现有客户端

✅ **接口文档**
- 自动生成OpenAPI/Swagger文档
- 访问：http://localhost:8001/docs

### 5.2 API规范

```
GET    /api/v1/chat          # 获取对话历史
POST   /api/v1/chat          # 发送消息
GET    /api/v1/tasks         # 获取任务列表
POST   /api/v1/tasks         # 提交任务
GET    /api/v1/tasks/{id}    # 查询任务状态
GET    /api/v1/monitor       # 系统监控
GET    /api/v1/agents        # Agent信息
```

### 5.3 前端集成示例

```javascript
// 前端只需调用统一接口，无需关心后端实现

  method: 'POST',
  headers: {'Content-Type': 'application/json'},
  body: JSON.stringify({message: '你好', user_id: 1})
});
```

## 6. RAG检索优化：主题搜索 + 知识摘要

### 6.1 核心增强

✅ **主题搜索**
- 按知识主题分类检索
- 提高搜索结果相关性

✅ **知识摘要**
- 自动生成搜索结果摘要
- 快速了解内容要点

✅ **增强过滤**
- 支持多维度筛选
- 时间、类型、来源等

✅ **智能排序**
- 综合相关性、权威性、时效性
- 提供更精准的搜索结果

### 6.2 搜索流程

```
用户查询 → 主题识别 → 向量检索 → 主题过滤 
         → 生成摘要 → 综合排序 → 返回结果
```

### 6.3 实现位置

**文件：** `core/rag_search_engine.py`

### 6.4 使用示例

```python
from core.rag_search_engine import RAGSearchEngine

engine = RAGSearchEngine()

# 主题搜索
results = engine.search_by_topic(
    query="人工智能",
    topic="technology",  # 限定主题
    top_k=5
)

# 带摘要的搜索
results_with_summary = engine.search_with_summary(
    query="机器学习算法",
    generate_summary=True  # 自动生成摘要
)

for result in results_with_summary:
    print(f"标题: {result['title']}")
    print(f"摘要: {result['summary']}")
    print(f"相关性: {result['relevance_score']:.2f}")
```
