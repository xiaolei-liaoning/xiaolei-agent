## 🤖 机器学习预测

### 什么是机器学习预测？

机器学习预测功能可以帮助您：
- **预测数值**：比如预测房价、销售额、股票价格等
- **预测类别**：比如预测客户是否会购买、邮件是否为垃圾邮件等
- **时间序列预测**：比如预测未来几天的销售趋势、用户增长等

### 如何使用？

#### 1. 准备数据
创建一个CSV文件，包含您的数据。例如：

```csv
feature1,feature2,feature3,target
1,2,3,6
2,4,6,12
3,6,9,18
4,8,12,24
5,10,15,30
```

#### 2. 选择预测类型

**回归预测**（预测数值）
```
用户: "预测target列的值"
系统: 自动使用回归模型预测数值
```

**分类预测**（预测类别）
```
用户: "预测category列的类别"
系统: 自动使用分类模型预测类别
```

**时间序列预测**（预测趋势）
```
用户: "预测未来10天的sales"
系统: 自动分析趋势并预测未来值
```

#### 3. 查看结果

系统会返回：
- 模型性能指标（MSE、RMSE、准确率）
- 特征重要性分析
- 预测结果
- 趋势分析（时间序列）

### 实际应用场景

#### 场景1：销售预测
```
用户: "预测下个月的销售额"
系统: 
  - 分析历史销售数据
  - 识别重要影响因素
  - 预测未来销售额
  - 提供趋势分析
```

#### 场景2：客户流失预测
```
用户: "预测哪些客户可能会流失"
系统:
  - 分析客户行为数据
  - 识别流失风险因素
  - 预测流失概率
  - 提供特征重要性
```

#### 场景3：库存预测
```
用户: "预测未来7天的库存需求"
系统:
  - 分析历史库存数据
  - 预测未来需求量
  - 提供趋势分析
  - 帮助优化库存管理
```

### 💡 使用技巧

1. **数据质量很重要**
   - 确保数据完整，尽量减少缺失值
   - 处理异常值和错误数据
   - 特征越多，预测效果越好

2. **选择合适的预测类型**
   - 目标是数值 → 回归预测
   - 目标是类别 → 分类预测
   - 需要预测未来趋势 → 时间序列预测

3. **关注特征重要性**
   - 了解哪些因素影响最大
   - 优化数据收集策略
   - 改进预测模型

## 📝 批量翻译

### 什么是批量翻译？

批量翻译功能可以：
- 一次性翻译多个文本
- 自动保存翻译历史
- 提供详细的翻译报告
- 处理翻译错误

### 如何使用？

#### 1. 准备文本
准备需要翻译的文本列表，例如：
```python
texts = ['你好', '世界', 'Python', '机器学习']
```

#### 2. 执行批量翻译
```
用户: "批量翻译：你好、世界、Python、机器学习"
系统: 自动识别并批量翻译
```

#### 3. 查看结果
系统会返回：
- 翻译总数
- 成功数量
- 失败数量
- 每条翻译的结果
- 错误信息（如果有）

### 实际应用场景

#### 场景1：文档翻译
```
用户: "批量翻译这个文档的所有段落"
系统:
  - 逐段翻译文档
  - 保存翻译历史
  - 提供翻译报告
  - 记录错误项
```

#### 场景2：多语言网站
```
用户: "批量翻译网站标题和描述"
系统:
  - 翻译所有文本
  - 保存到历史
  - 支持后续查询
  - 便于版本管理
```

#### 场景3：产品信息
```
用户: "批量翻译产品名称和描述"
系统:
  - 高效处理大量文本
  - 保持翻译一致性
  - 自动保存记录
  - 便于审核和修改
```

### 💡 使用技巧

1. **合理设置批次大小**
   - 单次翻译不要太多（建议<100条）
   - 注意API调用频率限制
   - 避免超时和错误

2. **利用历史记录**
   - 自动保存，无需手动记录
   - 便于后续查询和复用
   - 支持历史对比

3. **处理错误**
   - 查看错误报告
   - 分析失败原因
   - 重新翻译失败项

## 📚 翻译历史

翻译历史功能可以：
- 自动保存每次翻译
- 按日期组织记录
- 支持历史查询
- 便于追溯和复用

### 如何使用？

#### 1. 自动保存
每次翻译后，系统会自动保存到历史记录：
```
用户: "翻译：你好世界"
系统: 
  - 执行翻译
  - 自动保存到历史
  - 返回翻译结果
```

#### 2. 查询历史
```
用户: "查看最近7天的翻译历史"
系统:
  - 查询历史记录
  - 按时间排序
  - 显示详细信息
```

#### 3. 历史记录内容
每条记录包含：
- 时间戳
- 原文
- 译文
- 源语言
- 目标语言

### 实际应用场景

用户: "查看昨天翻译的所有内容"
系统:
  - 显示历史记录

```

#### 场景2：术语一致性

用户: "查看'机器学习'的所有翻译"
系统:
  - 搜索历史记录

#### 场景3：质量检查

用户: "查看最近一周的翻译"
系统:
  - 显示所有翻译
  - 便于质量检查

```

### 💡 使用技巧

1. **定期查看历史**
   - 检查翻译质量
   - 发现潜在问题
   - 优化翻译策略

2. **利用历史记录**
   - 复用已翻译内容
   - 保持术语一致性
   - 提高翻译效率

3. **管理历史文件**
   - 定期清理旧记录

## ❓ 常见问题

### 机器学习预测

**Q: 预测准确率不高怎么办？**
A: 
- 检查数据质量
- 增加更多特征
- 尝试不同的预测类型
- 查看特征重要性分析

**Q: 可以预测多长时间的未来？**
A: 
- 时间序列预测支持自定义步数
- 建议不要预测太远（通常<30步）
- 预测越远，准确性越低

**Q: 需要多少数据才能预测？**
A: 
- 至少需要几十条数据
- 数据越多，预测越准确
- 建议至少100条以上

### 批量翻译

**Q: 一次可以翻译多少条？**
A: 
- 理论上没有限制
- 建议单次<100条
- 注意API调用频率

**Q: 翻译失败怎么办？**
A: 
- 查看错误报告
- 检查文本格式
- 重新翻译失败项

**Q: 历史记录会占用多少空间？**
A: 
- 每天最多100条
- 每条约1KB
- 建议定期清理

### 翻译历史

**Q: 历史记录保存多久？**
A: 
- 按日期分文件存储
- 可以查询任意天数
- 建议定期清理

**Q: 如何删除历史记录？**
A: 
- 手动删除历史文件
- 位置：`skills/translator/history/`
- 文件名：`translation_history_YYYYMMDD.json`

**Q: 可以导出历史记录吗？**
A: 
- 历史记录是JSON格式
- 可以直接复制使用
- 便于备份和迁移
