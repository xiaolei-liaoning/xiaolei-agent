- 关键词数量: 20 → 7 (减少65%)

## 📈 对智能搜索的影响

### 为什么得分可能略降？

**现象**: 优化后搜索得分从 0.596 → 0.484

**原因分析**:

1. **关键词更精准，但数量减少**
   ```python
   # 优化前: 使用5个关键词
   keywords = ["帮我查询", "查询一下", "北京的", "今天的天", "天气情况"]
   # 其中3个是噪声词，但仍然能匹配到文档中的"北京"、"天气"
   
   # 优化后: 使用5个高质量关键词
   keywords = ["查询", "一下", "北京", "天气情况", "知道"]
   # 虽然质量高，但"一下"、"知道"在文档中出现频率低
   ```

2. **匹配策略变化**
   ```python
   # 优化前: 噪声词也能匹配
   "北京的" in "北京今日天气预报" → True (部分匹配)
   
   # 优化后: 精确匹配
   "北京" in "北京今日天气预报" → True (完全匹配)
   "天气情况" in "北京今日天气预报" → False (未出现"情况")
   ```

3. **TF-IDF计算更严格**
   - 优化后使用jieba的TF-IDF，分数计算更准确
   - 不再给低频噪声词高分

### 如何进一步提升搜索得分？

#### 方案1: 调整关键词选择策略

```python
# 当前: 使用前5个
keywords = [kw.word for kw in result.keywords[:5]]

# 优化: 只使用高分关键词 (>1.0)
keywords = [kw.word for kw in result.keywords if kw.score > 1.0]

# 或者: 按类别选择
keywords = (
    [kw.word for kw in result.keywords if kw.category == "地点"][:1] +
    [kw.word for kw in result.keywords if kw.category == "对象"][:2] +
    [kw.word for kw in result.keywords if kw.category == "动作"][:1]
)
```

#### 方案2: 增强文档库

```python
# 确保文档包含更多关键词变体
documents = [
    {
        "title": "北京今日天气情况预报",  # 添加"情况"
        "content": "北京今天的天气情况如下...",  # 包含完整短语
        "hierarchy_level": 1
    }
]
```

#### 方案3: 调整搜索权重

```python
# 在 search_engine.py 中

    "cosine": 0.6,      # 提高语义匹配权重
    "tfidf": 0.2,       # 降低词频权重
    "hierarchy": 0.2    
}
```

#### 方案4: 关键词扩展

```python
# 基于提取的关键词，生成相关词
base_keywords = ["北京", "天气"]
expanded_keywords = base_keywords + [
    "北京市",      # 同义词
```
