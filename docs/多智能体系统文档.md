# 多智能体系统详细技术文档

## 1. 架构设计

### 1.1 整体架构
- **架构模式**：【多角色独立智能体 + 内部并发分身】
- **部署模式**：单机多实例异步架构，不跨设备
- **核心组件**：BaseAgent基类、具体Agent实现、AgentScheduler调度中心
- **通信机制**：基于异步任务队列的消息传递

### 1.2 组件架构

#### 1.2.1 BaseAgent 基类
- **核心功能**：
  - 任务队列管理
  - 工作线程池管理
  - 任务状态跟踪
  - 人物SKILL加载与执行
- **关键方法**：
  - `start()`：启动Agent
  - `stop()`：停止Agent
  - `submit_task()`：提交任务
  - `_load_character_skills()`：加载人物SKILL
  - `_handle_character_task()`：处理人物SKILL任务

#### 1.2.2 具体Agent实现
- **CheckerAgent**：
  - 职责：网站状态检查、系统状态检查
  - 并发能力：最大5个工作线程
  - 任务类型：website、system

- **ScraperAgent**：
  - 职责：网站内容爬取、搜索
  - 并发能力：最大10个工作线程
  - 任务类型：website、search

- **VulnerabilityAgent**：
  - 职责：漏洞扫描、漏洞分析
  - 并发能力：最大5个工作线程
  - 任务类型：scan、analysis

- **SummarizerAgent**：
  - 职责：文本总结、报告总结
  - 并发能力：最大3个工作线程
  - 任务类型：text、report

- **PlanningAgent**：
  - 职责：智能任务规划、任务分解、执行协调
  - 并发能力：最大5个工作线程
  - 任务类型：planning、automation

- **HybridSkillAgent**：
  - 职责：技能管理、技能组合、工作流优化
  - 并发能力：最大8个工作线程
  - 任务类型：skill_management、skill_chain、workflow_optimization

#### 1.2.3 AgentScheduler 调度中心
- **核心功能**：
  - Agent生命周期管理
  - 任务分配与路由
  - 任务状态查询
- **任务映射**：
  - 维护任务类型到Agent类型的映射关系
  - 支持直接映射和关键词匹配

### 1.3 并发架构
- **线程模型**：每个Agent独立的线程池
- **任务队列**：每个Agent独立的异步队列
- **异步执行**：基于asyncio的非阻塞执行
- **负载均衡**：基于任务类型的智能分配

### 4.4 任务调度算法

- **负载均衡**：基于Agent当前负载

- **超时处理**：任务执行超时机制

### 4.5 人物SKILL加载机制
- **动态扫描**：运行时扫描skills/人物目录
- **模块导入**：使用importlib动态导入
- **实例化**：自动创建handler实例
- **缓存机制**：缓存已加载的人物SKILL

### 4.6 错误处理机制
- **异常捕获**：全局异常捕获
- **错误日志**：详细的错误日志
- **故障隔离**：一个任务失败不影响其他任务
- **重试机制**：支持任务重试

### 4.7 PlanningAgent 实现
- **核心功能**：
  - 任务分解：使用 TaskDecomposer 将复杂任务分解为子任务
  - 计划生成：根据分解结果生成执行计划
  - 任务执行：协调执行子任务
  - 结果汇总：整合子任务执行结果

- **工作原理**：
  1. 接收自然语言任务描述
  2. 使用 TaskDecomposer 分解任务
  3. 生成执行计划，考虑任务优先级和依赖关系
  4. 按计划执行子任务
  5. 汇总执行结果，生成报告

- **关键方法**：
  - `execute(task_description)`：执行复杂任务
  - `_generate_plan(decomposition_result)`：生成执行计划
  - `_execute_plan(plan)`：执行计划
  - `_summarize_results(results)`：汇总执行结果

### 4.8 HybridSkillAgent 实现
- **核心功能**：
  - 技能管理：动态加载和管理社区技能
  - 技能组合：创建和执行技能链
  - 工作流优化：分析和优化XML工作流
  - 技能匹配：根据任务描述匹配合适的技能

- **工作原理**：
  1. 初始化时扫描并加载 skills 目录下的所有技能
  2. 提供技能执行、技能链执行和工作流优化功能
  3. 支持技能的动态发现和调用
  4. 提供XML工作流的分析和优化

- **关键方法**：
  - `execute_skill(skill_name, **params)`：执行单个技能
  - `execute_skill_chain(skill_chain)`：执行技能链
  - `create_skill_chain(tasks)`：创建技能链
  - `optimize_xml_workflow(xml_content)`：优化XML工作流
