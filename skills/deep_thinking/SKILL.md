# 深度思考技能（基于 Deep Thinking Protocol）

## 技能信息
- **技能名称**: deep_thinking
- **技能描述**: 深度思考引擎，基于 Deep Thinking Protocol 5阶段框架，具备自主联网搜索能力，能够进行多步推理和自我反思

## 功能说明

### 核心特性
- **5阶段思考框架**: 问题理解 → 信息收集 → 方案设计 → 执行验证 → 反思优化
- **思考深度自动选择**: 
  - ⚡ Quick（快速模式）: 简单问题，< 5秒
  - 🔄 Standard（标准模式）: 常规任务，5-30秒
  - 🧠 Deep（深度模式）: 复杂问题，> 30秒
- **决策复杂度矩阵**: 根据任务复杂度和影响程度智能选择思考深度
- **质量标准验证**: 完整性、正确性、可行性、效率、可解释性
- **自主搜索**: 自动发现信息不足并进行网络搜索
- **实时信息**: 获取最新的网络信息
- **自我反思**: 验证答案正确性，自动纠错

### 使用场景
1. **获取最新信息**:
   - "2026年最新的人工智能发展趋势是什么？"
   - "今天的科技新闻有哪些重要内容？"

2. **深度分析**:
   - "分析一下当前的经济形势"
   - "深入探讨人工智能对就业的影响"

3. **研究问题**:
   - "研究一下如何提高学习效率"
   - "了解一下区块链技术的最新应用"

4. **复杂规划**:
   - "帮我规划一个完整的项目方案"
   - "分析这个问题并提供多个解决方案"

## 技术原理

### Deep Thinking Protocol 5阶段框架

1. **Phase 1: 问题理解**
   - 理解用户意图和需求
   - 识别约束条件
   - 定义成功标准
   - 设置任务边界

2. **Phase 2: 信息收集**
   - 检查短期记忆
   - 检索知识库
   - 评估信息需求
   - 执行自主搜索

3. **Phase 3: 方案设计**
   - 生成多个解决方案
   - 评估各方案优劣
   - 选择最优方案
   - 定义执行步骤
   - 准备备用方案

4. **Phase 4: 执行验证**
   - 实施方案
   - 验证结果
   - 检查点验证
   - 置信度评估

5. **Phase 5: 反思优化**
   - 反思答案质量
   - 多轮迭代优化
   - 提取关键经验
   - 记录可复用模式

### 决策复杂度矩阵

| 复杂度\影响 | 低影响 | 中影响 | 高影响 |
|------------|-------|-------|-------|
| **简单** | Quick | Quick | Standard |
| **中等** | Quick | Standard | Deep |
| **复杂** | Standard | Deep | Deep |

### 质量标准

- **完整性**: 所有子任务都被处理，没有隐含假设
- **正确性**: 逻辑一致，信息准确，工具使用正确
- **可行性**: 技术上可实现，在资源约束内
- **效率**: 思考时间与任务价值成正比
- **可解释性**: 能清楚说明决策理由

## API 使用

### Handler 方法

```python
from skills.deep_thinking.handler import get_deep_thinking_handler

handler = get_deep_thinking_handler()

# 基础使用（自动选择思考深度）
result = await handler.execute("分析一下人工智能的未来趋势", user_id=1)

# 指定思考深度
result = await handler.execute(
    "研究这个复杂问题", 
    user_id=1,
    depth="deep",  # "quick"/"standard"/"deep"
    show_thinking=True  # 是否显示思考过程
)

# 获取结果
print(result["reply"])  # 格式化的回复
print(result["result"]["final_answer"])  # 最终答案
```

### 参数说明

| 参数 | 类型 | 必需 | 默认值 | 说明 |
|------|------|------|--------|------|
| query | str | 是 | - | 用户查询内容 |
| user_id | int | 否 | 1 | 用户ID |
| depth | str | 否 | None | 强制使用的思考深度 ("quick"/"standard"/"deep") |
| show_thinking | bool | 否 | True | 是否在回复中显示思考过程 |

### 返回值说明

```python
{
    "success": bool,
    "result": {
        "final_answer": str,  # 最终答案
        "thinking_process": {  # 完整思考过程
            "phase1": {...},  # 问题理解
            "phase2": {...},  # 信息收集
            "phase3": {...},  # 方案设计
            "phase4": {...},  # 执行验证
            "phase5": {...},  # 反思优化
            "understanding": {...},
            "search_results": [...],
            ...
        },
        "thinking_depth": str,  # 使用的思考深度
        "quality_report": {...},  # 质量评估报告
        "elapsed_time": float,  # 耗时（秒）
        "protocol_used": bool  # 是否使用了新协议
    },
    "reply": str  # 格式化的回复文本
}
```

## 思考深度选择指南

### Quick 模式（⚡）
**适用场景**:
- 简单问题
- 已知答案
- 快速响应需求
- 低影响决策

**特点**:
- 跳过深度搜索
- 不执行多轮反思
- 快速返回结果

**示例**:
- "你好"
- "现在几点了"
- "简单计算"

### Standard 模式（🔄）
**适用场景**:
- 常规问题
- 需要一定分析
- 中等影响决策

**特点**:
- 执行标准5阶段流程
- 可能执行搜索
- 单轮反思优化

**示例**:
- "分析这个问题"
- "帮我找些资料"
- "提供一些建议"

### Deep 模式（🧠）
**适用场景**:
- 复杂问题
- 需要深度分析
- 高影响决策
- 多步推理

**特点**:
- 完整5阶段流程
- 全面搜索
- 多轮反思迭代（最多3次）
- 详细的质量评估

**示例**:
- "深入分析经济形势"
- "研究并规划完整方案"
- "解决这个复杂问题"

## 依赖项
- SerpAPI（可选，用于实时搜索）
- aiohttp（用于网络请求）
- LLM 后端（用于深度思考和推理）

## 注意事项
- 思考深度会根据问题复杂度自动选择，也可以手动指定
- 深度模式可能需要更长时间，但提供更高质量的答案
- 搜索功能需要网络连接
- 质量评分仅供参考，帮助理解答案的可靠程度
- 系统保留历史结果缓存，相同问题可能直接返回缓存结果
