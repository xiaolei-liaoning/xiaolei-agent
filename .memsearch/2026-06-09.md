# Tool Systems Analysis - 2026-06-09

## 三个项目工具系统对比

### gemini-cli (Google)

- **架构**: Builder-Invocation 两阶段模式
- **工具数**: 25+ (含 MCP/SDK/Discovered)
- **定义**: `DeclarativeTool` → `BaseDeclarativeTool` → 具体工具类
- **模型感知**: 不同模型族 (gemini-3/legacy) 返回不同 FunctionDeclaration Schema
- **调度**: Scheduler 批量调度 + PolicyEngine 规则引擎
- **安全**: Docker/Podman 沙箱、命令注入检测、路径边界检查、速率限制
- **独特功能**: Tail Call (工具请求后续调用)、wait_for_previous (LLM 控制并行/串行)、任务追踪器 (tracker_* 系列)、发现工具 (外部命令)
- **Hook**: BeforeTool/AfterTool 可拦截/修改/追加上下文

### opencode (TypeScript)

- **架构**: Effect-based 函数式 + AI SDK
- **工具数**: 16 内置 + MCP/Plugin
- **定义**: `Tool.define()` + Effect Schema → 自动生成 JSON Schema
- **依赖注入**: 通过 `yield*` 声明文件系统/LSP/HTTP 等依赖
- **安全**: Permission Service (allow/deny/ask 通配符规则)、external_directory 检查、AST Shell 分析 (tree-sitter)
- **独特功能**: 9 种模糊匹配编辑策略、LSP 集成 (编辑/读取通知)、BOM 感知文件操作、截断智能提示
- **扩展**: Plugin system + MCP + 文件自定义工具

### v2 (我们的项目)

- **架构**: dict + handler 函数
- **工具数**: 12 内置
- **定义**: 手动 JSON Schema dict + lambda handler
- **安全**: 无权限系统、无沙箱、无命令注入检测
- **缺失**: MCP 集成、Hook 系统、模型感知 Schema、Skill 系统
