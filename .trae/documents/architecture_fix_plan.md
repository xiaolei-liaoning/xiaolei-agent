# 架构设计缺陷修复计划

## 问题分析

### 1. 技能加载机制受限
- **问题**：技能只能从项目内的 `skills/` 目录加载，无法使用全局共享技能
- **影响**：无法复用 `.trae/skills/` 等全局技能仓库

### 2. 模块耦合度高
- **问题**：`skill_dispatcher.py` 硬编码大量关键词，与具体技能强耦合
- **影响**：添加新技能需修改核心代码

### 3. 配置管理问题
- **问题**：配置分散在多个文件中，缺乏统一管理
- **影响**：难以维护和扩展

## 解决方案

### 方案1：全局技能加载机制

**目标**：支持从多个目录加载技能（项目内 + 用户目录 + 全局目录）

**修改文件**：
- `core/skill_loader.py`（新建）- 技能加载器
- `core/skill_dispatcher.py` - 集成新的加载器

**实现步骤**：
1. 创建 `SkillLoader` 类，支持多目录扫描
2. 定义技能搜索路径优先级：
   - `./skills/`（项目目录）
   - `~/.xiaolongxia/skills/`（用户目录）
   - `~/.trae/skills/`（全局目录）
3. 支持技能热加载（文件监听）

### 方案2：外置关键词配置

**目标**：将关键词配置从代码中分离，支持动态加载

**修改文件**：
- `config/skill_keywords.yaml`（新建）- 关键词配置文件
- `core/skill_dispatcher.py` - 从配置文件加载关键词

**实现步骤**：
1. 创建 YAML 配置文件，定义技能名称、关键词、优先级
2. 修改 `SkillDispatcher` 从配置文件加载
3. 支持配置热加载

### 方案3：技能优先级机制

**目标**：当多个技能匹配时，根据优先级确定执行顺序

**修改文件**：
- `core/skill_dispatcher.py` - 增强匹配逻辑

**实现步骤**：
1. 在匹配结果中考虑优先级权重
2. 支持技能版本和评分机制
3. 优先选择萃取的技能

## 预期效果

| 指标 | 优化前 | 优化后 |
|------|--------|--------|
| 技能加载来源 | 单一目录 | 多目录（项目/用户/全局） |
| 关键词配置 | 硬编码 | 外置YAML文件 |
| 热加载 | 不支持 | 支持（文件监听） |
| 优先级管理 | 固定 | 可配置 |
| 扩展性 | 差 | 好 |

## 风险评估

| 风险 | 描述 | 应对措施 |
|------|------|----------|
| 配置文件格式错误 | YAML解析失败 | 添加配置验证和错误处理 |
| 技能名称冲突 | 不同目录有同名技能 | 按优先级选择，记录冲突日志 |
| 性能影响 | 多目录扫描耗时 | 缓存已加载技能，定期刷新 |
