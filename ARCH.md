# 两种模式架构对比

## 模式1：multi_agent_v2 — 固定角色多Agent系统

```
┌──────────────────────────────────────────────────────────────────────┐
│                        IntelligentScheduler                          │
│              (协作策略: pipeline / masterslave / review               │
│                / auction / hybrid)                                   │
└──────────────────────────┬───────────────────────────────────────────┘
                           │ 调度
                           ▼
┌──────────────────────────────────────────────────────────────────────┐
│                        GlobalContextCenter                           │
│                        (全局共享上下文)                                │
└──────────────────────────────────────────────────────────────────────┘
                           │
     ┌─────────────────────┼─────────────────────┐
     │                     │                     │
     ▼                     ▼                     ▼
┌────────────┐     ┌────────────┐     ┌──────────────┐
│ MasterAgent│     │ WorkerAgent│     │ReviewerAgent │
│ (任务分解)  │     │ (工具执行)  │     │ (质量评审)    │
│ (结果聚合)  │     │            │     │              │
└──────┬─────┘     └──────┬─────┘     └──────┬───────┘
       │                  │                  │
       └──────────────────┼──────────────────┘
                          │
                    ┌─────▼──────┐
                    │   Mind     │ ← 每个 Agent 都有自己的 Mind
                    │ ┌────────┐ │    (思考 → 执行 → 反思)
                    │ │ Memory │ │
                    │ │ System │ │
                    │ └────────┘ │
                    └────────────┘
                          │
                    ┌─────▼──────┐
                    │  LLM Router│ ← 真实 ZhipuAI / OpenAI
                    └────────────┘
```

**数据流（一次任务执行）：**

```
用户请求
   │
   ▼
MasterAgent.think()        ← LLM 理解任务
MasterAgent.decompose()    ← LLM 拆成子任务
   │
   ▼ 通过 CommunicationCenter 分配
   │
WorkerAgent.think()        ← LLM 理解子任务
WorkerAgent.execute()      ← 调用 ToolRegistry / SkillRegistry
WorkerAgent.reflect()      ← LLM 反思执行结果
   │
   ▼ 结果回到 MasterAgent
   │
ReviewerAgent.review()     ← LLM 评审质量
   │
   ▼
MasterAgent.aggregate()    ← 汇总输出
```

**Agent 间通信：**

```
         publish("task_completed")
         ┌─────────────────────┐
         │                     │
         │   CommunicationCenter
         │   (消息总线)         │
         │                     │
    subscribe ◄──────────► publish / broadcast
         │                     │
    send_direct           request/response
```

## 模式2：agency_agent.py — 动态Agent系统

### 单Agent模式

```
┌──────────────────────────────────────────────────┐
│                  run_agent()                      │
│                                                  │
│  ┌──────────┐   ┌──────────┐   ┌──────────────┐  │
│  │AGENCY.md │   │ToolReg.  │   │ SkillReg.    │  │
│  │(人格/规则)│   │(内置工具) │   │ (MCP技能)    │  │
│  └─────┬────┘   └────┬─────┘   └──────┬───────┘  │
│        │              │                │          │
│        ▼              ▼                ▼          │
│  ┌──────────────────────────────────────────┐     │
│  │          System Prompt                    │     │
│  │  AGENCY.md + 可用工具列表 + 调用规则       │     │
│  └──────────────────┬───────────────────────┘     │
│                     │                              │
│  ┌──────────────────▼───────────────────────┐     │
│  │          LLM Router                       │     │
│  │                                          │     │
│  │  ┌──────────────┐     ┌──────────────┐   │     │
│  │  │  直接回答    │     │  调用工具    │   │     │
│  │  │  ← 返回文本  │     │  → 循环继续  │   │     │
│  │  └──────────────┘     └──────────────┘   │     │
│  └──────────────────────────────────────────┘     │
└──────────────────────────────────────────────────┘

循环过程：
  LLM 回答 → 检查是否工具调用 → 否 → 返回给用户
                                → 是 → 执行工具 → 可选反思 → 结果回 LLM → 继续

失败时反问：
  LLM 超时/异常 → QuestionRegistry → 用户回答 → retry → 重试
                                                → cancel → 取消
                                                → proceed/超时 → 返回错误信息
```

### 团队模式 (run_team) — 队长 + 队员 + 消息总线

```
run_team("分析这个项目")
   │
   │ 1. 队长(LLM)分解任务、分配角色
   ▼
┌───────────────────────────────────────────────────┐
│  「数据分析师: 爬取数据」                         │
│  「统计分析员: 清洗和分析」                       │
│  「报告撰写员: 生成报告」                         │
└───────────────────────┬───────────────────────────┘
                        │
  2. 初始化 SharedContext（消息总线）
                        │
┌───────────────────────────────────────────────────┐
│  SharedContext (共同记忆 + 按频道消息总线)          │
│                                                    │
│  频道[指挥]     频道[成果]      频道[告警]         │
│  队长→开始      队员1→完成      队员2→数据异常     │
│  队长→指令      队员2→结果      ...                │
└───────────────────────┬───────────────────────────┘
                        │
  3. 队员并行执行 (asyncio.gather)
   ┌──────────┬──────────┼──────────┐
   │          │          │          │
   ▼          ▼          ▼          ▼
┌──────┐  ┌──────┐  ┌──────┐  ┌──────┐
│队员1  │  │队员2  │  │队员3  │  │...   │  ← 各调 run_agent()
│分析师 │  │统计员 │  │报告员 │  │      │     (带反问+反思)
└──┬───┘  └──┬───┘  └──┬───┘  └──────┘
   │         │         │
   │  $summary / $report → 总线
   └────┬────┴─────┬───┘
        │         │
        ▼         ▼
   SharedContext 更新（队员发现→共同记忆）
        │
        │ 4. 队长(LLM)汇总，带全部频道信息
        ▼
   最终回答给用户
```

## 对比总结

| 维度 | 模式1 (multi_agent_v2) | 模式2 (agency_agent.py) |
|------|----------------------|------------------------|
| 规模 | 47 文件，~14000 行 | 1 文件，~300 行 |
| Agent 定义 | BaseAgent 类继承，有状态/记忆/心智 | 无 Agent 类，纯函数 |
| 协作粒度 | 5 种策略，Agent 间可自由通信 | 并行执行 + 汇总 |
| 共享状态 | GlobalContextCenter + 消息总线 | SharedContext 对象 |
| 持久化 | AgentPool + 健康检查 + 持久化 | 无，用完就丢 |
| 适用场景 | 需要稳定角色分工的长流程 | 临时任务，快速出结果 |
